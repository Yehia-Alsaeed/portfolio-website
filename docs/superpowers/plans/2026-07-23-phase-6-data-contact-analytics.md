# Phase 6 Data, Contact, and Analytics Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Executor:** Claude Code. Do not implement until Yehia explicitly asks you to execute this plan.

**Goal:** Add durable contact persistence and privacy-respecting first-party analytics on isolated Neon databases without making public page rendering or navigation depend on the backend.

**Architecture:** Keep every public route static/ISR and put all database access behind focused server-only Drizzle query modules. The contact Server Action and analytics route share validated request facts, HMAC identity, and atomic Neon rate limits; a secured daily Vercel cron rolls raw events into dimension rows before enforcing 90-day retention. Vercel-managed Neon branches isolate previews, and Resend remains implemented but disabled.

**Tech Stack:** Existing Next.js 16.2.10, React 19.2.7, TypeScript 6.0.3, Tailwind CSS 4.3.3, Vitest, Testing Library, Playwright, axe, Lighthouse, GitHub Actions, and Vercel Hobby. Add `drizzle-orm@0.45.2`, `@neondatabase/serverless@1.1.0`, `@vercel/functions@3.7.5`, and dev dependency `drizzle-kit@0.31.10`.

- [ ] **Step 3: Implement one atomic, idempotent maintenance batch**

```ts
const todayUtc = "YYYY-MM-DD";
const retentionStartUtc = todayUtc minus 90 calendar days;
```
